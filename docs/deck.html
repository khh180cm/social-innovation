300명을 매주 — 보고 · 판단 · 조치

미취업 청년 일경험 사업의 운영자가, 끝나고 나서가 아니라 매주 미리 위험을 보고 손쓸 수 있게 돕는 도구입니다.
문제

운영은 항상 한 박자 늦는다

  • 출석·수당이 주 단위로 느리게 집계 → 8주 끝나야 미달자 확인 (이미 늦음)
  • 참여자 300명 — 누가 / 왜 / 무엇을 이 안 보임
  • 개입해도 효과 측정·학습이 안 됨
현장 운영자의 고통은 정보가 없어서가 아니라, 너무 늦게, 흩어져서 온다는 겁니다.

솔루션

한 화면, 3단계

예측 필터  →  12유형 매트릭스  →  AI 맞춤 솔루션

페이지 이동 없이 한 보드에서

탭을 넘나드는 대시보드가 아니라, 한 화면에서 끝나는 운영 보드입니다. 클릭 두세 번으로 위험군 → 개인 → 조치까지.
1단계

예측 필터 — 느린 데이터를 이긴다

  • 업로드된 주차만 관측, 남은 주는 예측
  • Discounted Beta-Binomial + 코호트 prior
    • conjugate → 매주 산수 한 줄로 온라인 업데이트
    • prior 풀링이 콜드스타트(1·2주)를 받침
    • λ=0.8 망각이 5주차 추세 시프트 반영
    • posterior로 미지급 조기경보 (신뢰구간)
켤레사전분포라 가벼우면서 콜드스타트·추세변화·조기경보를 한 모델이 다 받습니다. 1주차엔 신뢰구간이 넓고, 주차가 쌓이며 좁아집니다.
2단계

12유형 매트릭스

  • 구간(긴급/위험/주의/안정) × 경향(적극/보통/하락) = 12분류
  • 셀 클릭 = 즉시 명단 필터 (이동 없음)
  • “즉시 조치 대상 N/300명” 한눈에
300명을 12칸으로 압축. 빨간 칸(긴급)부터 봅니다. 같은 미달이라도 올라오는 중인지 떨어지는 중인지 나눠 다르게 대응.
3단계

AI 맞춤 솔루션

  • 개인 드릴다운 → Claude claude-opus-4-8 가
  • 주차별 피드백 + 유형 + 예측 + 직전 조치 효과를 종합
  • 차등 조치 2~3개 + 메시지 초안 생성
  • 실행/승인은 운영자 (human-in-the-loop)
단순 요약이 아니라, 직전 주 조치가 먹혔는지 먼저 평가하고 안 먹혔으면 다른 접근을 냅니다. 결정은 사람이.

AI 활용을 두 가지로 나눕니다. 만들 때 AI를 어떻게 썼나, 제품 안에서 AI가 어떻게 동작하나. 둘 다 PRD·프로세스·규칙을 명시적으로 설계.
A. 만들 때

오케스트레이션 — PRD · 프로세스 · 규칙

  • PRD — 발표 3단계가 곧 요구사항. "무엇을·왜"를 먼저 글로 정의 → 위임
  • 프로세스 — 문제정의 → 근거 리서치 → 설계 → 반복 구현 → 검증
  • 규칙 — 정직성 / 근거기반 / 교차검증 / 정직한 결과수용 / 데이터 스코프
막연히 '만들어줘'가 아니라 각 단계를 요구사항으로 정의해 위임하고 결과를 제가 판단. 작업 내내 규칙을 강제 — 특히 인과 단정 금지, 과장 금지.
A. 만들 때

근거 리서치 — 멀티에이전트 워크플로우

  • 에이전트들이 청년고용 ALMP · 이탈 조기경보 · 자가진화 루프 문헌 검색
  • 실제로 열어본 출처만 인용 → 별도 에이전트가 재검증

Card·Kluve·Weber 2010OECD Lyche 2010 J-PAL RCTJITAI (Nahum-Shani 2018)

AI가 그럴듯한 출처를 지어내지 않도록, 실제로 열어 확인한 문헌만 인용하게 강제하고 다른 에이전트가 검증. 솔루션 7원칙은 전부 여기서 나왔습니다.
B. 제품 안

규칙 = 시스템 프롬프트

AI를 규칙 · 구조 · 게이트에 가둬 신뢰성 확보

  • 규칙 = 검증된 7원칙 (구조적 장벽 우선 · 세그먼트 차등 · 타이밍 · 인과 단정 금지 · human-in-the-loop …)
  • 구조 = 강제 도구 호출(tool_choice+JSON 스키마) → 출력 항상 일정
  • 게이트 = 실행/승인은 운영자
제품 안 AI의 'rule'은 곧 시스템 프롬프트. 임의 조언이 아니라 검증된 원칙을 강제하고, 스키마로 출력 형태를 보장하고, 최종 결정권은 사람에게.
신뢰성

왜 믿을 수 있나

  • 정직성 — "효과 있다" ❌ → "할 수 있다/권한다" ⭕, 더미는 로직 검증용
  • 교차검증 — 핵심 수치 독립 재확인 (TS 계산 ↔ Python 분석 일치)
  • 정직한 결과 수용 — 백테스트에서 진 구간도 포장 없이 공개
  • human-in-the-loop — AI는 제안, 사람이 결정
AI 도구의 최대 리스크는 그럴듯하지만 틀린 출력. 그걸 구조적으로 막았습니다. 우리 모델이 진 구간도 숨기지 않았습니다.
적용

적용 계획

  • 빌드: 더미 CSV(300명×8주)로 로직 검증
  • 적용: 실제 일경험 사업(예: 소셜WE 아트브릿지 2기)에 매주 업로드 → 보드
  • 확장: predict→act→measure→learn 루프로 개입 효과 누적 학습
지금은 더미로 로직을 증명했고, 실제 사업에선 매주 출석만 올리면 같은 보드가 돕습니다. 다음 단계는 개입 효과를 학습하는 자가진화 루프.
마무리

AI를 단계별 파트너로 쓰되, 제품 안의 AI는 검증된 규칙 + 강제 구조 + 사람 승인 안에서만 동작하게 설계해 — 과장과 환각을 구조적으로 차단했다.
