# PRD — 미래내일 일경험 운영 판단 도구

## 0. 한 줄
미취업 청년 일경험 사업 **운영자**가 300명 참여자의 상태를 **빠르게 보고·판단·조치**하도록 돕는 AI 도구.

## 1. 문제
- 출석·수당이 **주 단위로 느리게** 집계된다. 운영자는 8주가 끝나야 미달자를 안다 → **이미 늦음**.
- 참여자 300명을 사람이 일일이 챙길 수 없다. **누가 위험한지, 왜, 무엇을 할지**가 한눈에 안 보인다.
- 개입을 해도 **효과가 있었는지** 측정·학습되지 않는다.

## 2. 사용자 & 4가지 핵심 판단
운영자(사업 담당자)가 매주 묻는 질문 = 도구가 답해야 할 것:

| # | 판단 | 도구의 답 |
|---|---|---|
| 1 | 누가 **위험**한가? | 긴급도(긴급/위험/주의/안정) 자동 분류 |
| 2 | **왜** 그런가? | 주차별 멀티소스 피드백(교사·외부기관·알림톡·구글폼) |
| 3 | **무엇을** 할까? | AI 차등 맞춤 조치 + 메시지 초안 |
| 4 | 끝까지 가면 어떻게 되나? | 미지급 확률·기대 수당 예측(신뢰구간) |

## 3. 기능 (3단계, 원 페이지)
1. **예측 필터** — 업로드된 주차만 관측, 남은 주는 예측. Discounted Beta-Binomial + 코호트 prior(콜드스타트 풀링), λ=0.8 망각(추세 시프트 반영), posterior로 미지급 조기경보.
2. **유형 12분류 매트릭스** — 구간(4) × 경향(적극/보통/하락, 3). 셀 클릭 = 명단 필터(페이지 이동 없음).
3. **AI 맞춤 솔루션** — 드릴다운 → Claude(`claude-opus-4-8`)가 근거 7원칙 + 직전 조치 효과를 반영해 조치 제안. 실행/승인은 운영자(human-in-the-loop).

## 4. 데이터
- 빌드: 더미 `participants.csv`(300명 × 8주, 직무훈련/일경험 두 트랙).
- 멀티소스 주차 피드백은 출결과 상관되게 결정론적 생성(seed=id).
- 실 적용 대상(내러티브): 소셜WE 아트브릿지 2기 등.

## 5. 비기능 (원칙)
- **정직성** — 인과 단정 금지, 더미는 로직 검증용, 과장 금지.
- **human-in-the-loop** — AI는 제안, 사람이 결정.
- **근거 기반** — 설계 주장은 검증된 출처에 연결([AI_METHODOLOGY.md](AI_METHODOLOGY.md) 참조).
- **프라이버시** — API 키 서버 전용, 브라우저 노출 0.

## 6. 범위 밖 (YAGNI)
실시간 연동, 다중 페이지/네비게이션, 권한·로그인, 실제 수당 지급 처리. — MVP는 "판단을 돕는 한 화면".
