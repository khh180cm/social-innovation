# AI를 어떻게 활용했는가 — 방법론 (PRD · 프로세스 · 규칙)

> 이 문서는 발표에서 자주 받는 질문 — *"구현뿐 아니라 AI를 어떻게 활용했는지, PRD·프로세스·규칙을 어떻게 설계해서 AI와 작업했는지"* — 에 대한 답이다.

AI 활용은 **두 층위**로 나뉜다.

| 층위 | 무엇 | 한 줄 |
|---|---|---|
| **A. 제품을 만들 때 AI** | 이 도구 자체를 AI와 함께 설계·구현 | "구현은 AI에게, 판단은 사람에게" |
| **B. 제품 안에서 AI** | 도구가 운영자에게 맞춤 조치를 제안 (3단계) | "AI는 규칙·구조·게이트 안에서만 동작" |

---

## A. 제품을 만들 때 — 오케스트레이션 방법론

AI를 "코드 자판기"로 쓰지 않았다. **근거 리서치 → 설계 → 구현 → 검증**의 각 단계 파트너로 썼다.

### 1) PRD (요구사항을 먼저 글로 정의)
구현 전에 "무엇을, 왜"를 명시했다. 발표 3단계가 곧 PRD였다:

1. **예측 필터** — 피드백이 느린 시스템에서 1·2주로 3주를 어떻게 예측?
2. **참여자 유형 12분류** — 구간(긴급/위험/주의/안정) × 경향(적극/보통/하락)
3. **멀티소스 맞춤 솔루션** — 교사·외부기관·알림톡·구글폼 + 직전 조치 효과 → AI 제안

→ 각 단계를 AI에게 "요구사항"으로 주고 구현을 위임, 결과를 사람이 판단.

### 2) 프로세스 (정의된 작업 흐름)
```
문제 정의(브레인스토밍)
  → 근거 리서치 (멀티에이전트 워크플로우: 권위 문헌 검색 → 인용 검증 → 종합)
  → 설계 (문제정의·솔루션 방향)
  → 반복 구현 (화면을 보며 완성)
  → 검증 (교차검증 · 백테스트 · 타입체크)
```
- **근거 리서치를 AI 워크플로우로 병렬화**했다. 예: 청년고용 ALMP·이탈 조기경보·발달장애 지원고용·자가진화 루프(JITAI/contextual bandit) 문헌을 에이전트들이 검색하고, **실제로 열어본 출처만(opened=true)** 인용하도록 강제, 별도 에이전트가 인용을 재검증했다. → Card·Kluve·Weber(2010), Kluve(IZA 10263), OECD Lyche(2010), J-PAL, JITAI(Nahum-Shani 2018), Oralytics RL(2024) 등 **검증된 출처만** 설계 근거로 사용.

### 3) 규칙 (AI 협업의 가드레일)
작업 내내 강제한 규칙:

| 규칙 | 내용 |
|---|---|
| **정직성** | 인과 단정 금지. 더미 데이터는 "로직 검증용"으로만. "효과 있다" ❌ → "할 수 있다/권한다" ⭕ |
| **근거 기반** | 모든 설계 주장은 검증된 출처에 연결. 추정은 "추정"으로 라벨 |
| **교차검증** | 핵심 수치는 독립 방식으로 재확인 (예: TypeScript 계산 ↔ Python 분석이 지급률 81.0%·미지급 73셀 일치) |
| **정직한 결과 수용** | 백테스트에서 우리 필터가 안정 구간에선 단순평균보다 낮았다 → **포장하지 않고 사실대로** 제시(5주 시프트에선 우위) |
| **데이터 스코프** | 더미 CSV로 빌드, 실제 프로그램(소셜WE아트브릿지)은 발표 내러티브로만 |

## B. 제품 안에서 — AI(Claude)를 어떻게 설계했나 (3단계)

3단계 "AI 맞춤 솔루션"은 AI를 **규칙·구조·게이트** 안에 가둬 신뢰성을 확보했다.

### PRD (AI가 할 일의 정의)
> 한 참여자의 **주차별 멀티소스 피드백 + 유형(긴급도×경향) + 예측(미지급확률·기대수당) + 직전 주 조치 효과**를 종합해, 운영자가 바로 실행할 **차등 맞춤 조치 2~3개 + 메시지 초안**을 제안한다. 먼저 직전 조치가 효과 있었는지 평가하고, 없었으면 다른 접근을 제시한다.

### 프로세스 정의 (자가진화 루프)
`predict → act → measure → learn` — 예측하고(1·2단계), 조치를 제안·실행(3단계), 다음 주 데이터로 효과를 측정, 효과 없으면 사람(상담)으로 에스컬레이션. (JITAI·rapid-cycle evaluation 계보)

### 규칙 = 시스템 프롬프트 (검증된 근거 7원칙)
AI의 "rule"은 곧 시스템 프롬프트다. 임의 조언이 아니라 **검증된 원칙**을 강제했다:

1. 동기부여 문구가 아니라 **구조적 장벽 해소**(교통비·잡정보) 우선 (J-PAL)
2. **유형별 차등** — 같은 개입도 세그먼트별 역효과 (J-PAL India RCT)
3. **타이밍** — 초기 광범위·저비용 / 후기 표적·1:1 (OECD Lyche)
4. **포기 위험** = 금전 독려 무의미 → 비금전·완주 재설정
5. **천장** = 출석 독려 대신 성과·안착 전환
6. **푸시** = "X일 더 나오면 +OO원" 구체 넛지
7. **human-in-the-loop** — AI는 제안만, 실행·승인은 운영자. 인과 단정 금지
